# Chain Reaction (연쇄 폭발)

---

## 문제 설명

핵물리 연구소에서 두 팀의 과학자들이 연쇄 폭발 반응 실험을 시작했습니다.
6×6 격자의 반응로에 구슬을 하나씩 투입하며, 구슬이 일정 수 이상 쌓이면 **폭발**이 일어나 인접 칸으로 퍼져나갑니다.
더 많은 구슬을 보유한 팀이 실험의 승자가 됩니다.

### 게임 규칙

* **Rule 1:** 플레이어는 자신의 턴에 **빈 칸** 또는 **자신의 구슬이 있는 칸**에 구슬 1개를 추가합니다.
* **Rule 2:** 칸에 구슬이 해당 칸의 **용량(capacity)** 이상 쌓이면 폭발합니다.
  * 폭발 시 그 칸에서 용량만큼 구슬이 제거되고, 상하좌우 인접 칸에 각 1개씩 전파됩니다.
  * 전파된 구슬은 **현재 플레이어의 색**으로 변환됩니다 (상대 구슬도 전환).
  * 인접 칸도 용량을 초과하면 연쇄 폭발이 발생합니다.
* **Rule 3:** 각 칸의 용량은 해당 칸의 인접 칸 수와 같습니다.
  * 모서리 칸: 2, 가장자리 칸: 3, 내부 칸: 4
* **Rule 4:** 양쪽 플레이어가 모두 최소 1번 이상 행동한 후, 상대방의 구슬이 모두 사라지면 승리합니다.
* **Rule 5:** 플레이어당 최대 25턴이 주어지며, 25턴이 지나면 총 구슬 수가 많은 플레이어가 승리합니다.

> **목표:** 연쇄 폭발을 전략적으로 활용해 상대방의 구슬을 모두 제거하거나, 더 많은 구슬을 보유하세요.

---

## 보드 초기 상태

게임 시작 시 각 플레이어의 구슬 4개가 랜덤하게 배치됩니다.
`INIT` 메시지로 초기 배치를 전달받습니다.

---

## 입/출력 형식

참가자의 프로그램은 한 줄씩 입력을 받아 각 명령에 따라 동작해야 합니다.

| 커맨드 | 입력 형식 | 설명 |
| :--- | :--- | :--- |
| **READY** | `READY (FIRST\|SECOND)` | 선공(FIRST) 또는 후공(SECOND) 정보. `OK`를 3초 이내에 **출력해야 합니다.** |
| **INIT** | `INIT r0 r1 r2 r3 r4 r5` | 6×6 보드 초기 상태. 각 ri는 6자리 문자열입니다.<br>`0`=빈 칸, `1~3`=선공 구슬 1~3개, `4~6`=후공 구슬 1~3개 |
| **TIME** | `TIME t1 t2` | 내 남은 시간(t1)과 상대 남은 시간(t2). 구슬을 놓을 칸의 좌표를 `r c` 형식으로 **출력해야 합니다.**<br>- 보드 좌상단은 `(0,0)`, 우하단은 `(5,5)`<br>- 초기 시간: 10,000ms |
| **OPP** | `OPP r c t` | 상대가 구슬을 놓은 칸 좌표(r,c)와 사용 시간(t). (출력 불필요) |
| **FINISH** | `FINISH` | 게임 종료 신호. 프로그램을 정상 종료해야 합니다. (출력 불필요) |

> :warning: **주의:** 모든 출력 후에는 반드시 버퍼를 flush해야 합니다 (`endl` 또는 `flush`).

---

## 좌표 규칙

- 행(row): 위에서부터 0~5
- 열(col): 왼쪽부터 0~5
- 유효한 행동: `r c` (0 ≤ r ≤ 5, 0 ≤ c ≤ 5, 빈 칸 또는 내 구슬이 있는 칸)
