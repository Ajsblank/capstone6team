# 초능력 윷놀이 (Hardcore Real-Rules)
**NYPC CODE BATTLE · 온라인 라운드 연습문제**

---

## 📖 문제 설명
각 플레이어는 4개의 말을 조종하며, 무작위로 배정받은 **'윷을 다스리는 힘' 1개**와 **'말을 다스리는 힘' 1개**를 활용하여 상대방보다 먼저 4개의 말을 결승점으로 통과시켜야 합니다. 단 한 번의 출력 실수나 반칙(사용 불가능한 스킬 사용 등)은 **즉각적인 패배**로 이어집니다.

### 🗺️ 맵 구조 및 말의 상태
* **-1번 (대기 상태):** 초기에 말은 보드 위 어디에도 위치하지 않습니다. (강제 귀가나 폭탄에 당할 경우 이 상태로 돌아옵니다.)
* **0번 (결승점):** 결승점입니다. 이곳에 도달한 말은 골인 처리되어 더 이상 움직일 수 없습니다.
* **1번 ~ 19번:** 바깥쪽 일반 트랙
* **21번 ~ 27번:** 교차로 지름길 (21, 22: 우상단 / 24, 25: 좌상단 / 23: 정중앙 / 26, 27: 결승 방향)

### 🐎 이동 및 윷놀이 규칙
* **출발:** `-1`번(대기) 상태에서 '도(1)'가 나오면 `1`번 칸으로, '개(2)'가 나오면 `2`번 칸으로 출전합니다. 대기 상태에서 '빽도(-1)'로는 출전할 수 없습니다.
* **빽도 꼼수:** `1`번 칸에 위치할 때 '빽도(-1)'를 이동하면 뒤로 한 칸 이동하여 즉시 `0`번(결승점)으로 골인합니다.
* **지름길:** 모서리 5번, 10번 칸에서 **출발**할 경우 무조건 지름길(21번, 24번)로 꺾어집니다.
* **업기 & 잡기:** 이동을 마친 내 말이 '내 다른 말'과 만나면 업고 다닙니다. '상대 말'과 만나면 상대를 **대기 상태(-1번)**로 강제 귀환(잡기)시키고, 주사위를 한 번 더 굴릴 기회를 얻습니다.

## 🔮 초능력 스킬 및 턴 규칙

### [스킬 발동 제약 조건]
1. 내 턴에 **주사위를 한 번이라도 굴렸다면 스킬을 사용할 수 없습니다.** (즉, 턴을 시작하자마자 굴리기 전에 써야 합니다.)
2. '윷 스킬'과 '말 스킬'은 한 턴에 **동시에 사용할 수 없습니다.** (한 턴에 최대 1번만 스킬 사용 가능)
3. 각 스킬은 게임 전체를 통틀어 딱 1번만 사용할 수 있습니다.

### [윷을 다스리는 힘] (이동 수치 변환)
* 대상 지정이 필요 없으므로 출력 시 파라미터를 생략하고 `SKILL <ID>` 형태로만 출력합니다.

| ID | 스킬명 | 효과 |
| :-- | :-- | :-- |
| 1 | 개와 돼지의 시간 | 윷을 굴리지 않고 이번 턴에 **`1(도)`와 `2(개)`를 모두** 얻습니다. |
| 2 | 100 마력 | 윷을 굴리지 않고 `3(걸)`을 얻습니다. |
| 3 | 후진 기어 | 윷을 굴리지 않고 `빽도(-1)`를 얻습니다. |
| 4 | 디스 이스 더블더블| 스킬 사용 직후 굴린 윷의 결과를 2배로 얻습니다. |
| 5 | 모 아니면 도 | 스킬 사용 직후 굴린 윷의 결과를 확률(50%)에 따라 `1` 또는 `5`를 얻습니다. |
| 6 | 마이더스의 손 | 스킬 사용 직후 굴린 윷의 결과를 확률(50%)에 따라 `4` 또는 `5`를 얻습니다. |
| 7 | 테이크 아웃 | 이번 턴에 굴린 윷 수치를 즉시 쓰지 않고 '저장'해두었다가 다음 턴에 씁니다. |
| 8 | 백스텝 | 스킬 사용 직후 굴린 윷의 결과만큼 '뒤로(-)' 이동하는 수치를 얻습니다, 말이 없으면 사용할 수 없습니다. |

### [말을 다스리는 힘] (보드 직접 개입)
* 특정 대상 지정이 필요하므로 `SKILL <ID> <Param1> <Param2>` 형식으로 출력합니다.

| ID | 스킬명 | 효과 | Param1 (P1) | Param2 (P2) |
| :-- | :-- | :-- | :-- | :-- |
| 9 | 강제 귀가 | 상대 말을 대기 상태(`-1`번)로 귀환 | 무시됨: `0` | 상대 말 인덱스 (`0~3`) |
| 10 | 업고 가실게요 | 대기 상태(`-1`번) 내 말을 지정한 위치로 소환 | 내 말 인덱스 (`0~3`) | `0` |
| 11 | 자리 좀 바꿔주세요| 내 말과 상대 말의 위치 서로 교환 | 내 말 인덱스 (`0~3`) | 상대 말 인덱스 (`0~3`) |
| 12 | 제 앞에 놔주세요 | 상대 말을 내 말 '1칸 앞'으로 소환 | 내 말 인덱스 (`0~3`) | 상대 말 인덱스 (`0~3`) |
| 13 | 백허그 | 내 말을 상대 말 '1칸 뒤'로 텔레포트 | 내 말 인덱스 (`0~3`) | 상대 말 인덱스 (`0~3`) |
| 14 | 밥상 뒤집기 | 필드 모든 말 무작위 강제 재배치 | `0` | `0` |
| 15 | 결승점이 코앞이네| 내 말을 `0`번(골인)으로 텔레포트 | 내 말 인덱스 (`0~3`) | `0` |
| 16 | 폭발물 주의 | 지정 노드에 지뢰 설치 | 무시됨: `0` | 노드 번호 (`1~19, 21~27`) |

## ⌨️ 입/출력 형식 (인터랙션 프로토콜)

프로그램이 행동을 취할 때마다, 시스템은 동기화를 위해 **즉각적인 결과(`RESULT`)**를 반환합니다. 

### [시스템 ➡️ 프로그램] 입력 명령어
* **`READY ...`** (게임 매칭, `OK` 출력 필요)
* **`INIT <My_윷스킬> <My_말스킬> <Opp_윷스킬> <Opp_말스킬>`** (시작 시 1회)
* **`TIME <my_time> <opp_time> <can_roll> <rolled_turn> <skill_turn> <ms1> <ms2> <os1> <os2> <bomb> <M0~M3> <O0~O3> <yut_cnt> [yuts...]`**
  * 턴 시작 시 현재 전체 보드 상태와 남은 윷 목록을 전달합니다.
* **`OPP <count>`\n`[로그1]`\n`[로그2]`...** * 상대가 턴에 수행한 명령어 원본 로그 전송 (턴 시작 전)
* **`ROLL_RESULT <yut_value>`** (윷 굴린 결과)
* **`SKILL_RESULT <유형> <결과값들...>`**
  * `YUT <count> <yut1>...` : (스킬 1,2,3,5,6) 윷을 획득함
  * `BOARD <M0~M3> <O0~O3>` : (스킬 9~16) 스킬로 인해 보드가 재배치됨
  * `OK` : (스킬 4,7,8) 정상 접수됨
* **`MOVE_RESULT <caught: 0 or 1> <M0~M3> <O0~O3>`** * 이동 후 상대 잡기 성공 여부(`1`이면 잡음) 및 변동된 전체 보드 상태 전송

### [프로그램 ➡️ 시스템] 반응 출력
1. **`ROLL`** : 윷을 던집니다. (이후 `ROLL_RESULT` 입력 대기)
2. **`SKILL <ID> [P1] [P2]`** : 스킬을 사용합니다. (이후 `SKILL_RESULT` 입력 대기)
3. **`MOVE <말_인덱스> <윷수치>`** : 말을 이동합니다. (이후 `MOVE_RESULT` 입력 대기)
4. **`PASS`** : 턴을 종료합니다.
