# 사과 게임

---

## 문제 설명

아주대학교 캠퍼스 한편에 있는 사과 과수원에서, 아주대학교의 상징 **치토**들이 사과 따기 대결을 펼치기로 했습니다.
과수원의 사과나무들은 10행 17열로 빼곡히 심어져 있고, 각 나무에는 **1개에서 9개**의 사과가 열려 있습니다.
두 치토는 번갈아 가며 구역을 정해 사과를 따고, 더 많은 사과나무 구역을 차지한 쪽이 승리합니다.

### 게임 규칙

* **Rule 1:** 플레이어는 자신의 턴에 과수원에서 하나의 직사각형 구역을 선택합니다.
* **Rule 2:** 선택한 구역 안에 열린 사과의 총 개수가 정확히 **10개**인 경우, 해당 구역의 사과를 모두 따고 그 구역을 점령합니다.
    * 이때, 구역 안에 상대방이 이미 점령한 칸이 포함되어 있다면 해당 칸의 소유권도 빼앗을 수 있습니다.
    * 또한, 선택한 직사각형의 네 변에는 각각 사과가 남아 있는 나무가 최소 한 그루 이상 인접해 있어야 합니다.
* **Rule 3:** 선택 가능한 구역이 없거나 전략적으로 선택을 하지 않으려 한다면, 선택을 생략하고 턴을 넘길 수 있습니다.
* **Rule 4:** 두 플레이어가 연속으로 턴을 넘기면 게임이 종료됩니다. 종료 시점에 더 많은 구역을 점령한 플레이어가 승리합니다.

> **목표:** 선공 치토 또는 후공 치토가 되어, 과수원에서 더 많은 사과나무 구역을 차지할 수 있는 프로그램을 작성하세요.

---

## 입/출력 형식

참가자의 프로그램은 한 줄씩 입력을 받아 각 명령에 따라 동작해야 합니다. 출력이 필요한 명령은 아래 표에서 강조 표시되어 있습니다.

| 커맨드 | 입력 형식 | 설명 |
| :--- | :--- | :--- |
| **READY** | `READY (FIRST\|SECOND)` | 플레이어의 선공/후공 정보를 알려주는 명령입니다. 입력 준비 완료를 의미하는 `OK`를 3초 이내에 **출력해야 합니다.** |
| **INIT** | `INIT ...` | 과수원 초기 상태를 알려주는 명령입니다. `...`에는 10개의 문자열이 주어지며, 각 문자열은 17자리 숫자로 구성되어 있습니다. 각 숫자는 1이상 9이하의 정수이며, 해당 위치의 사과나무에 열린 **사과 개수**를 의미합니다. (출력 불필요) |
| **TIME** | `TIME t1 t2` | 현재까지 각 플레이어의 남은 시간을 알려주는 명령입니다. `t1`은 내 프로그램의 남은 시간, `t2`는 상대의 남은 시간입니다.<br>이 명령을 받으면 사과를 딸 직사각형 구역의 좌표를 `r1 c1 r2 c2` 형식으로 **출력해야 합니다.**<br>- 과수원의 왼쪽 위 좌표는 `(0,0)`, 오른쪽 아래는 `(9,16)`입니다.<br>- 패스할 경우 `-1 -1 -1 -1`을 출력합니다.<br>- 처음 주어지는 시간은 10,000(ms)입니다. |
| **OPP** | `OPP r1 c1 r2 c2 t` | 상대방이 딴 구역 정보를 알려주는 명령입니다. 상대방이 선택한 구역 좌표와 해당 턴에 사용한 시간 `t`가 주어집니다. 상대가 패스한 경우에는 `-1 -1 -1 -1`이 주어집니다. (출력 불필요) |
| **FINISH** | `FINISH` | 게임 종료를 알리는 명령입니다. 이 명령을 받으면 프로그램은 정상적으로 종료되어야 합니다. (출력 불필요) |

> :warning: **주의사항:** 모든 출력 이후에는 반드시 버퍼를 flush해야 합니다. 버퍼를 flush하지 않은 경우 출력이 제대로 되지 않아 TLE(시간 초과)가 발생할 수 있습니다.
